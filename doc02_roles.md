# 02 運営とユーザーの役割分担

## 当日の全体像

```
[運営（GM）]                         [参加者（36武将）]
   │                                       │
   ├─ rv_gm_dashboard.html                 ├─ rv_player_app.html（個人スマホ）
   │   ・ルール設定（GPS半径・判定方式）         │   ・武将選択・暗証番号入力
   │   ・同期バックエンド接続（GAS/Firebase）   │   ・GPS占領申請
   │   ・承認キューで申請を承認               │   ・ヒント回答入力
   │   ・武将割当（QRリンク生成）             │   ・現地写真提出
   │   ・陣所ピンのドラッグ修正              │   ・疾風の采配（移動距離計測）
   │                                       │   ・LINE公式に「戦況」「登録」送信
   └─ rv_kunitori_board25.html             │
       ・全体盤の監視                      └─ rv_kunitori_board25.html
       ・?admin= 付きURLで占領・初期化        ・地図で陣所を見る（観戦モード）
                                           ・Kunitori.html で戦況を見る
```

## URL の使い分け

| URL | 誰に渡すか | 機能差 |
|---|---|---|
| `https://art-crea.github.io/RVcamp2026/rv_kunitori_board25.html` | 参加者全員 | 観戦のみ。占領ボタン非表示 |
| `…?admin=honjin827`（←変更後のトークン） | **運営のみ** | 手動占領・初期化・地図切替・陣所編集 |
| `rv_player_app.html?room=RV827&busho=3&pin=1021` | 武将ごとのQR | 武将3番として接続（GMが生成） |
| `index.html` | 運営 | 検証HUB |

## 参加者フロー

1. GM本陣「参戦リンクを作る」→ QR生成 → 武将カードに印刷
2. 開戦前：スマホで武将画面のQRを読む → 武将確認 → 暗証番号入力 → 出陣
3. 陣所に到着 → GPS確認 → ヒント回答入力（or 写真撮影）→「申請」ボタン
4. GM本陣の承認キューに届く → 承認 → 盤面が即時更新
5. LINE公式に「戦況」と送ると現在スコアが返ってくる
